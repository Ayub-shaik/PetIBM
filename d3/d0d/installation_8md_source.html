3 ## Dependencies (last tested)
4 
5 **Required**:
6 
7 * GNU C++ compiler g++ (4.9.2, 5.4.0, and 7.2.1 have been tested)
8 * [PETSc](https://www.mcs.anl.gov/petsc/) (3.8+) with HDF5 enabled
9 * MPI: OpenMPI, MPICH, or Intel MPI
10 * [yaml-cpp](https://github.com/jbeder/yaml-cpp) (0.6.2)
11 * [gtest](https://github.com/google/googletest) (1.7.0)
12 
13 **Optional for GPU linear solvers**:
14 
15 * [AmgX](https://github.com/NVIDIA/AMGX) (2.0.0.130-opensource)
16 * [AmgXWrapper](https://github.com/barbagroup/AmgXWrapper) (1.4)
17 
18 **Optional for pre- and post-processing Python scripts**:
19 
20 * Python (3.6+)
21 * NumPy (1.12.1)
22 * h5py (2.7.0)
23 * Matplotlib (2.0.2)
24 
25 **Note**:
26 
27 * MPI can be either installed during [PETSc configuration](#petsc) or installed explicitly by users.
28 * [yaml-cpp](https://github.com/jbeder/yaml-cpp),
29  [gtest](https://github.com/google/googletest), and
30  [AmgXWrapper](https://github.com/barbagroup/AmgXWrapper) can be automatically installed during PetIBM configuration or explicitly installed by users in advance.
31 * PetIBM has been tested on:
32  * Ubuntu 16.04 with g++-5.4, and PETSc-3.11.2
33  * MacOS Sierra with g++-6.0, and PETSc-3.8.2
34  * Arch Linux with g++-7.2, and PETSc-3.8.2
35 * PetIBM has also been tested on the following HPC systems:
36  * [GW ColonialOne](https://colonialone.gwu.edu/)
37  * [Titan](https://www.olcf.ornl.gov/titan/) at ORNL

41 ## GNU C++ Compiler (`g++`)
42 
43 On Ubuntu, install `g++` using the following command:
44 
45  sudo apt-get install g++
46 
47 Check the version of G++ installed:
48 
49  g++ --version
50 
51 On Mac OS X, `g++` can be installed on Mac OS X via XCode or [Homebrew](brew.sh).

55 ## PETSc
56 
57 PetIBM relies on the data structures and parallel routines of the PETSc library.
58 
59 Here, we provide the command-line instructions to install PETSc-3.11.3 in debugging mode and/or optimized mode.
60 The debug mode is recommended during development, while the optimized mode should be used for production runs.
61 
62 Get and unpack PETSc:
63 
64 ```shell
65 cd $HOME/sfw
66 mkdir -p petsc/3.11.3
67 wget http://ftp.mcs.anl.gov/pub/petsc/release-snapshots/petsc-lite-3.11.3.tar.gz
68 tar -xvf petsc-lite-3.11.3.tar.gz -C petsc/3.11.3 --strip-components=1
69 cd petsc/3.11.3
70 ```
71 
72 Configure and build an debugging version of PETSc:
73 
74 ```shell
75 export PETSC_DIR=$HOME/sfw/petsc/3.11.3
76 export PETSC_ARCH=linux-dbg
77 ./configure --PETSC_DIR=$PETSC_DIR --PETSC_ARCH=$PETSC_ARCH \
78  --with-cc=gcc \
79  --with-cxx=g++ \
80  --with-fc=gfortran \
81  --COPTFLAGS="-O0" \
82  --CXXOPTFLAGS="-O0" \
83  --FOPTFLAGS="-O0" \
84  --with-debugging=1 \
85  --download-fblaslapack \
86  --download-openmpi \
87  --download-hdf5 \
88  --download-hypre \
89  --download-ptscotch \
90  --download-metis \
91  --download-parmetis \
92  --download-superlu_dist
93 make all
94 make test
95 ```
96 
97 If you do not have a Fortran compiler ready, you use the following configuration:
98 
99 ```shell
100 ./configure --PETSC_ARCH=$PETSC_ARCH \
101  --with-cc=gcc \
102  --with-cxx=g++ \
103  --COPTFLAGS="-O0" \
104  --CXXOPTFLAGS="-O0" \
105  --with-debugging=1 \
106  --download-f2cblaslapack \
107  --download-openmpi \
108  --download-hdf5 \
109  --download-hypre \
110  --download-ptscotch \
111  --download-metis \
112  --download-parmetis \
113  --download-superlu_dist
114 ```
115 
116 Configure and build an optimized version of PETSc:
117 
118 ```shell
119 export PETSC_DIR=$HOME/sfw/petsc/3.11.3
120 export PETSC_ARCH=linux-opt
121 ./configure --PETSC_DIR=$PETSC_DIR --PETSC_ARCH=$PETSC_ARCH \
122  --with-cc=gcc \
123  --with-cxx=g++ \
124  --with-fc=gfortran \
125  --COPTFLAGS="-O3" \
126  --CXXOPTFLAGS="-O3" \
127  --FOPTFLAGS="-O3" \
128  --with-debugging=0 \
129  --download-fblaslapack \
130  --download-openmpi \
131  --download-hdf5 \
132  --download-hypre \
133  --download-ptscotch \
134  --download-metis \
135  --download-parmetis \
136  --download-superlu_dist
137 make all
138 make test
139 ```
140 
141 When running the code on an external cluster, make sure that you compile PETSc with the MPI that has been configured to work with the cluster.
142 Hence, **do not use the `--download-openmpi` flag**, but instead point to the folder with the MPI compilers and executables using the `--with-mpi-dir` flag.
143 If BLAS and LAPACK are already installed in the system, you can point to the libraries using the `--with-blas-lib` and `--with-lapack-lib` flags.
144 
145 [Detailed instructions](http://www.mcs.anl.gov/petsc/documentation/installation.html) with more options to customize your installation can be found on the PETSc website.
146 Run `./configure --help` in the PETSc root directory to list all the available configure flags.
147 
148 The PETSc Users Manual and the Manual Pages can be found on their
149 [documentation page](http://www.mcs.anl.gov/petsc/documentation/index.html).

153 ## yaml-cpp
154 
155 [yaml-cpp](https://github.com/jbeder/yaml-cpp) is a YAML parser in C++, used in PetIBM to parse the input configuration file.
156 
157 The PetIBM configuration script gives the possibility to use a previously installed version of yaml-cpp (with the configuration argument `--with-yamlcpp-dir=<path>`, or `--with-yamlcpp-include=<path>` and `--with-yamlcpp-lib=<path>`) or to download and install yaml-cpp-0.6.2 (with `--enable-yamlcpp`).

161 ## gtest
162 
163 We use Google's C++ test framework [gtest](https://github.com/google/googletest) to run unit-tests after compiling PetIBM.
164 When configuring PetIBM, you can either use a previously installed version of gtest and provide the path of the directory of the gtest installation with `--with-gtest-dir=<path>` (or using `--with-gtest-include=<path>` and `--with-gtest-lib=<path>`) or request to download and install gtest-1.7.0 with `--enable-gtest`.

168 ## PetIBM
169 
170 Create a local copy of the PetIBM repository:
171 
172 ```shell
173 cd $HOME/sfw
174 mkdir petibm && cd petibm
175 git clone https://github.com/barbagroup/PetIBM.git
176 export PETIBM_DIR=$HOME/sfw/petibm/PetIBM
177 ```
178 
179 You can set the environment variable `PETIBM_DIR` to your `$HOME/.bashrc` or `$HOME/.bash_profile` files by adding the line:
180 
181 ```shell
182 export PETIBM_DIR=$HOME/sfw/petibm/PetIBM
183 ```
184 
185 and restart you terminal or source the file:
186 
187 ```shell
188 source $HOME/.bashrc
189 ```
190 
191 Note: if you are using C shell, use the `setenv` command instead of `export`.
192 
193 Configure and build PetIBM with PETSc in debugging mode:
194 
195 ```shell
196 PETSC_DIR=$HOME/sfw/petsc/3.11.3
197 PETSC_ARCH=linux-dbg
198 mkdir petibm-linux-dbg && cd petibm-linux-dbg
199 $PETIBM_DIR/configure \
200  --prefix=$HOME/sfw/petibm/petibm-linux-dbg \
201  CXX=$PETSC_DIR/$PETSC_ARCH/bin/mpicxx \
202  CXXFLAGS="-g -O0 -Wall -Wno-deprecated -std=c++14" \
203  --with-petsc-dir=$PETSC_DIR \
204  --with-petsc-arch=$PETSC_ARCH \
205  --with-yamlcpp-dir=<path> \
206  --with-gtest-dir=<path>
207 make all
208 make check
209 make install
210 ```
211 
212 If yaml-cpp and/or gtest are not available, you can request installation at configuration time using:
213 
214 ```shell
215 $PETIBM_DIR/configure \
216  --prefix=$HOME/sfw/petibm/petibm-linux-dbg \
217  CXX=$PETSC_DIR/$PETSC_ARCH/bin/mpicxx \
218  CXXFLAGS="-g -O0 -Wall -Wno-deprecated -std=c++14" \
219  --enable-yamlcpp \
220  --enable-gtest
221 ```
222 
223 For production runs, you may want to use the optimized build of PETSc:
224 
225 ```shell
226 PETSC_DIR=$HOME/sfw/petsc/3.11.3
227 PETSC_ARCH=linux-opt
228 mkdir petibm-linux-opt && cd petibm-linux-opt
229 $PETIBM_DIR/configure \
230  --prefix=$HOME/sfw/petibm/petibm-linux-opt \
231  CXX=$PETSC_DIR/$PETSC_ARCH/bin/mpicxx \
232  CXXFLAGS="-O3 -Wall -Wno-deprecated -std=c++14" \
233  --with-petsc-dir=$PETSC_DIR \
234  --with-petsc-arch=$PETSC_ARCH \
235  --with-yamlcpp-dir=<path> \
236  --with-gtest-dir=<path>
237 make all
238 make check
239 make install
240 ```
241 
242 Et voila!
243 You can now add the `bin` directory (that contains the executables) to you PATH environment variable:
244 
245 ```shell
246 export PATH=$HOME/sfw/petibm/petibm-linux-opt/bin:$PATH
247 ```

264 ## Optional: using NVIDIA AmgX to solve linear systems on multiple GPUs
265 
266 PetIBM implements the possibility to solve linear systems on CUDA-capable GPU devices using the NVIDIA [AmgX](https://github.com/NVIDIA/AMGX) library.
267 
268 To solve linear systems on multiple GPU devices with PetIBM, the user must install CUDA and AmgX before building PetIBM and use the configuration arguments `--with-cuda-dir=<path>` for CUDA and `--with-amgx-dir=<path>` (or `--with-amgx-include=<path>` and `--with-amgx-lib=<path>`) for AmgX.
269 
270 To handle the data conversion between PETSc and AmgX, we use our in-house wrapper, called [AmgXWrapper](https://github.com/barbagroup/AmgXWrapper).
271 When configuring PetIBM, you can either use a previously installed version of AmgXWrapper and provide the path of the directory of the AmgXWrapper installation with `--with-amgxwrapper-dir=<path>` (or using `--with-amgxwrapper-include=<path>` and `--with-amgxwrapper-lib=<path>`) or request to download and install AmgXWrapper-1.4 with `--enable-amgxwrapper`.
272 
273 For example, to build an optimized version of PetIBM with CUDA, AmgX, and AmgXWrapper:
274 
275 ```shell
276 PETSC_DIR=$HOME/sfw/petsc/3.11.3
277 PETSC_ARCH=linux-opt
278 mkdir petibm-linux-opt && cd petibm-linux-opt
279 $PETIBM_DIR/configure \
280  --prefix=$HOME/sfw/petibm/petibm-linux-opt \
281  CXX=$PETSC_DIR/$PETSC_ARCH/bin/mpicxx \
282  CXXFLAGS="-O3 -Wall -Wno-deprecated -std=c++14" \
283  --with-petsc-dir=$PETSC_DIR \
284  --with-petsc-arch=$PETSC_ARCH \
285  --with-yamlcpp-dir=<path> \
286  --with-gtest-dir=<path> \
287  --with-cuda-dir=<path> \
288  --with-amgx-dir=<path> \
289  --with-amgxwrapper-dir=<path>
290 make all
291 make check
292 make install
293 ```
