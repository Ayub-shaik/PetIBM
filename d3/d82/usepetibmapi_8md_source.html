1 # Use PetIBM API
2 
3 In the installation path, PetIBM also provides a library and header files that have all necessary API to build a flow solver.
4 
5 ## API Documentation
6 
7 The API documentation can be found in the **Modules** section of the [Doxygen pages](https://barbagroup.github.io/PetIBM/modules.html).
8 The subsection of **PetIBM building blocks** shows details of the elements for building a flow solver.
9 The subsection of **Flow solvers and utilities** shows the flow solvers built on those building blocks.
10 
11 A local version of Doxygen pages can be created with the `doxygen` utility.
12 To do that, go into the `doc` folder of PetIBM, and run the `doxygen` command.
13 HTML pages will be created in the subfolder `html` of the `doc` directory.
14 Users can, then, open the main page of the manual, `html/index.html`, using their preferred browser.
15 
16 ## API usage examples
17 
18 The source code of the flow solvers mentioned in the API documentation may be unfriendly for beginners, so we provide other simplified solvers in the folder `$PETIBM_DIR/examples/api_examples`.
19 For example, `liddrivencavity2d` provides step-by-step instructions in the comments of the source code.
20 (See the README files in those folders for additional details.)
21 
22 ## Compiling and linking against PetIBM
23 
24 ### Header files
25 
26 Let `$PETIBM_INSTALL` be the installation path of PetIBM.
27 All header files for public PetIBM API are under `$PETIBM_INSTALL/include/petibm`, while those for third-party dependencies, if any, are under `$PETIBM_INSTALL/include`.
28 For example, if users configure PetIBM using the flag `--enable-amgxwrapper`, the header file `AmgXSolver.hpp` will be copied to `$PETIBM_INSTALL/include`.
29 
30 There are many headers of PetIBM, while only the top-level ones are normally
31 used in an application code:
32 
33 * `type.h`: useful data types in PetIBM,
34 * `parser.h`: functions to parse YAML input file,
35 * `mesh.h`: abstract mesh object and its factory functions,
36 * `boundary.h`: abstract boundary (and ghost points) object and its factory functions,
37 * `solution.h`: abstract solution object and its factory functions,
38 * `operators.h`: factory functions for creating operators (matrices),
39 * `linsolver.h`: abstract object for linear solvers and its factory functions,
40 * `bodypack.h`: abstract object for multi-body holders and its factory functions,
41 * `timeintegration.h`: abstract object for temporal integration and its factory functions.
42 
43 To compile with PetIBM headers, remember to have `-I$PETIBM_INSTALL/include` in the compilation flags.
44 Also, the path to PETSc headers is necessary, e.g., `-I$PETSC_DIR/PETSC_ARCH/include`.
45 
46 If users provide their own installations of third-party dependencies to build
47 PetIBM, remember to also include those paths.
48 For example, if users provide `--with-yamlcpp-dir=$YAMLCPP_DIR` during configuration, then they will have to add `-I$YAMLCPP_DIR/include` to the compilation flags.
49 
50 ### Library
51 
52 PetIBM provides a single library, either `libpetibm.a` (static library) or `libpetibm.so` (shared library) to link against.
53 The library is located in `$PETIBM_INSTALL/lib`; use `-L$PETIBM_INSTALL/lib -lpetibm` to link.
54 
55 There may be other libraries in `$PETIBM_INSTALL/lib`, depending on how the users configure PetIBM.
56 For example, if the users use the configuration flag `--enable-yamlcpp`, then, the yaml-cpp library will be installed along with the PetIBM library in `$PETIBM_INSTALL/lib`.
57 (Note: on some systems or OS, third-party libraries may be installed in `$PETIBM_INSTALL/lib64`.)
58 
59 Also, linking against PETSc library is necessary.
60 Flags for linking against PETSc can be found in the file `$PETSC_DIR/$PETSC_ARCH/lib/petsc/conf/petscvariables`.
61 (Look for the line starting with `PETSC_WITH_EXTERNAL_LIB = `.)
62 
63 Alternatively, users can let the PetIBM library automatically handle dependencies with the `libtool` program.
64 PetIBM is build using `libtool`, which records dependencies information in an abstract layer `$PETIBM_INSTALL/lib/libpetibm.la`.
65 Although users can directly look into the `.la` file, the proper way to use `libtool` is:
66 
67 ```
68 libtool --tag=CXX --mode=link $CXX -o <executable name> <object files> \
69  $LINKER_FLAGS $PETIBM_INSTALL/lib/libpetibm.la
70 ```
71 
72 or ,if compiling and linking at once,
73 
74 ```
75 libtool --tag=CXX --mode=link $CXX -o <executable name> <source files> \
76  $CXX_FLAGS $LINKER_FLAGS $PATH_TO_HEADERS $PETIBM_INSTALL/lib/libpetibm.la
77 ```
